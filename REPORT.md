# 1 ВВЕДЕНИЕ

Современные финансовые рынки характеризуются высокой динамикой и большими объёмами торгов различными финансовыми инструментами. Фьючерсы на валюту являются одним из важнейших инструментов срочного рынка, позволяющих хеджировать валютные риски и осуществлять спекулятивные операции.

Фьючерс представляет собой контракт (обязательство) на будущую поставку (продажу) заданного вида товара в оговорённый день исполнения. Контракт может многократно перепродаваться за период его действия. Текущая цена контракта определяется прогнозом цены товара на день исполнения контракта.

Для эффективной работы на рынке фьючерсов необходимы инструменты анализа исторических данных торгов, позволяющие выявлять тенденции и прогнозировать изменение цен.

Цель работы: создание программных средств для накопления и анализа данных торгов активами срочного рынка.

Объект разработки: биржа, осуществляющая операции с ценными бумагами.

Информационный базис: данные торгов фьючерсами на доллар США.

## 2.1 Исходные данные

Структура информации о торгах (файл F_USD.XLS):
- дата торгов;
- код фьючерса Фk;
- текущая цена фьючерса Fk (в руб. за доллар США);
- объём продаж в этот день Vk (млн руб.).

Структура информации о датах исполнения фьючерсов (файл DATAISP.XLS):
- код фьючерса Фk;
- дата исполнения фьючерса Tk.

В каждый торговый день продаётся несколько фьючерсов, различающихся датами исполнения.

## 2.3 Формула расчёта основного показателя

Основной контролируемый показатель — логарифм изменения цены фьючерса Fk(i) за два торговых дня:

```
xk(i) = ln{ Fk(i) / Fk(i-2) }
```

где xk(i) — логарифм изменения цены фьючерса Fk за два торговых дня;  
i — порядковый номер торгового дня;  
Fk(i) — цена фьючерса в день i;  
Fk(i-2) — цена фьючерса два торговых дня назад.

### 3.2.4 Связи между таблицами

Рисунок 1 — Схема связей между таблицами базы данных

```
┌─────────────┐       ┌──────────────────┐       ┌─────────────┐
│   futures   │       │   expirations    │       │   trades    │
├─────────────┤       ├──────────────────┤       ├─────────────┤
│ code (PK)   │◄──────│ future_code (FK) │       │ trade_date  │
│ name        │       │ expiry_date      │       │ future_code │◄─┐
└─────────────┘       └──────────────────┘       │ price       │  │
      ▲                                          │ contracts   │  │
      │                                          └─────────────┘  │
      └───────────────────────────────────────────────────────────┘
```

Ограничения целостности:
- код фьючерса должен соответствовать формату FUSD_MM_YY;
- месяц в коде (MM) должен быть от 01 до 12;
- цена должна быть больше 0;
- количество контрактов должно быть >= 0 или NULL;
- дата торгов не может быть позже даты исполнения;
- при удалении фьючерса каскадно удаляются все связанные записи.
